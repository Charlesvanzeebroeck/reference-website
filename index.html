Following a consistent naming convention for all filenames, paths, and folders is crucial to preventing confusion and making file management simpler. Use of lowercase letters, hyphens instead of spaces, keeping names short and descriptive, and avoiding special characters are some best practises for naming.

Commit messages should be concise and clear, describing the changes made in the commit, according to best practises. Additionally, it's crucial to start with a capital letter and end with a period, use the present tense, and write in the imperative mood. Additionally, commit messages ought to give enough background information so that others can comprehend the modifications made.

HTML (Hypertext Markup Language) is a markup language used for creating web pages. It consists of a series of tags that are used to structure and format the content on a web page.

HTML tags consist of a pair of angle brackets enclosing the tag name. Some tags also include attributes, which provide additional information about the tag.

Commonly used HTML tags/elements:

  • Headings: h1-h6 - used to create headings of different sizes
  • p - used to create paragraphs
  • Lists: ul, ol, dl - used to create unordered lists, ordered lists, and definition lists
  • a - used to create hyperlinks
  • img - used to insert images
  • q - used to create inline quotations
  • blockquote - used to create block quotations
  • cite - used to indicate the source of a quotation or reference
  • em - used to emphasize text
  • strong - used to indicate strong importance
  • b - used to indicate bold text
  • i - used to indicate italic text
  • small - used to indicate smaller text

Block elements are HTML elements that create a block-level box on the web page, taking up the full width of the available space. The list of block elements includes: html head body header nav main section article div aside footer These elements are used to structure the content on a web page and provide semantic meaning.

Accessibility is the practice of designing web pages and applications that are accessible to people with disabilities. This is important to ensure that everyone can access and use the web, regardless of their abilities. Some accessibility properties include: Landmark roles - used to indicate the purpose of specific regions on a web page Aria labels - used to provide alternative text for non-text elements Image alternative texts - used to provide alternative text for images for users with screen readers.

CSS (Cascading Style Sheets) is a styling language used to control the appearance of web pages. It is used to define the layout, colors, fonts, and other visual aspects of a web page.

To implement CSS to an HTML file, you can either use inline CSS, internal CSS, or external CSS. External CSS is the preferred method as it allows for better organization and separation of concerns. To link an external CSS file to an HTML file, use the following code in the head section of the HTML file: < link rel="stylesheet" href="style.css">

CSS property vs. value: CSS properties are used to define the style of an HTML element, while values are used to specify the specific styling details. For example: font-family: Arial; In this example, font-family is the property, and Arial is the value.
